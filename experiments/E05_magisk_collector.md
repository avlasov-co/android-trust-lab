# E05 Magisk Collector

## Purpose

Validate Magisk module lifecycle and root-side snapshot collection.

## Target

Rooted target with Android Trust Lab collector module installed.

## Preconditions

- Owned or lab target.
- Magisk installed by the researcher.
- Module package built from this repo.

## Controlled change

Enable optional read-only privileged collector.

## Expected signals

- module installation visible
- boot-time collection report generated
- manual action collection report generated
- output path exists
- uninstall cleans internal temporary files
- collection manifest and raw artifact produced
- host analyzer can normalize the raw artifact into a schema-compatible report
- adb observer and root observer may differ

## Collection procedure

1. Package module.
2. Install module on controlled target.
3. Reboot.
4. Pull boot-time raw artifact and collection manifest.
5. Run manual action.
6. Pull manual raw artifact and collection manifest.
7. Normalize raw artifacts with `trustlab normalize --observer root_collector`.
8. Validate normalized reports.
9. Diff adb-side and root-side observations.

## Artifacts produced

- root collector raw artifact
- collection manifest
- normalized root collector report generated by host analyzer
- validation output
- observer diff

The sample demonstrates how an on-device collection manifest points to a raw artifact that is normalized later by the host analyzer.

## Diff summary

Root-side observer visibility differs from adb-shell visibility. The generated diff represents observer-boundary evidence, not a device-integrity verdict.

## Limitations

The collector must remain read-only. It does not prove hardware-backed boot trust. The included sample is synthetic and intended to validate analyzer and report behavior.

## Status

Sample artifacts included. Physical-device validation is not claimed.
